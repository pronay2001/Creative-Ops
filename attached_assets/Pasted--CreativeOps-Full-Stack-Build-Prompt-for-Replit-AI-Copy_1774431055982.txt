## What to Build

### Phase 1: Backend Server (Express + PostgreSQL)

Replace `server.js` with a full Express backend.

**Database schema** (use `pg` package, not an ORM):

```sql
-- Users table (syncs with Keka, also seeded locally)
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  keka_id TEXT UNIQUE,
  name TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  role TEXT NOT NULL DEFAULT 'requester',
  skills TEXT[] DEFAULT '{}',
  capacity INTEGER DEFAULT 0,
  avatar_url TEXT,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Campaigns
CREATE TABLE campaigns (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  show TEXT,
  status TEXT DEFAULT 'active',
  description TEXT,
  created_date TIMESTAMPTZ DEFAULT NOW(),
  created_by TEXT REFERENCES users(id),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Requests (creative tasks)
CREATE TABLE requests (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  campaign_id TEXT REFERENCES campaigns(id),
  asset_type_id TEXT NOT NULL,
  department TEXT,
  platforms TEXT[] DEFAULT '{}',
  assigned_to TEXT REFERENCES users(id),
  status TEXT DEFAULT 'intake',
  priority TEXT DEFAULT 'medium',
  go_live_date DATE,
  internal_deadline DATE,
  brief JSONB DEFAULT '{}',
  created_date TIMESTAMPTZ DEFAULT NOW(),
  created_by TEXT REFERENCES users(id),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Deliverables (sub-items of a request)
CREATE TABLE deliverables (
  id TEXT PRIMARY KEY,
  request_id TEXT REFERENCES requests(id) ON DELETE CASCADE,
  asset_type_id TEXT NOT NULL,
  platforms TEXT[] DEFAULT '{}',
  assigned_to TEXT REFERENCES users(id),
  status TEXT DEFAULT 'intake',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Comments
CREATE TABLE comments (
  id SERIAL PRIMARY KEY,
  request_id TEXT REFERENCES requests(id) ON DELETE CASCADE,
  user_id TEXT REFERENCES users(id),
  text TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Activity log
CREATE TABLE activity_log (
  id SERIAL PRIMARY KEY,
  request_id TEXT REFERENCES requests(id) ON DELETE CASCADE,
  user_id TEXT REFERENCES users(id),
  action TEXT NOT NULL,
  details JSONB DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Timesheet entries
CREATE TABLE timesheet_entries (
  id SERIAL PRIMARY KEY,
  user_id TEXT REFERENCES users(id),
  request_id TEXT REFERENCES requests(id),
  date DATE NOT NULL,
  hours NUMERIC(4,2) DEFAULT 0,
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Knowledge base entries (per campaign)
CREATE TABLE knowledge_entries (
  id SERIAL PRIMARY KEY,
  campaign_id TEXT REFERENCES campaigns(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  type TEXT,
  content TEXT,
  created_by TEXT REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Content schedule (per campaign)
CREATE TABLE content_schedule (
  id SERIAL PRIMARY KEY,
  campaign_id TEXT REFERENCES campaigns(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  platform TEXT,
  scheduled_date DATE,
  status TEXT DEFAULT 'planned',
  assigned_to TEXT REFERENCES users(id),
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Phase 4: Keka HR Integration

**Authentication**: Keka uses OAuth 2.0 with a custom grant type.
```
POST https://{KEKA_COMPANY}.keka.com/connect/token
Content-Type: application/x-www-form-urlencoded

grant_type=kekaapi
scope=kekaapi
client_id={KEKA_CLIENT_ID}
client_secret={KEKA_CLIENT_SECRET}
api_key={KEKA_API_KEY}
```

Response: `{ "access_token": "...", "expires_in": 3600, "token_type": "Bearer" }`

**Get employees**:
```
GET https://{KEKA_COMPANY}.keka.com/api/v1/hris/employees
Authorization: Bearer {access_token}
```

Response includes: `displayName`, `firstName`, `lastName`, `email`, `employeeNumber`, `department.name`, `jobTitle.name`, `isActive`

**Build**:
- `services/keka.js` - Keka API client with token caching (tokens expire in 1 hour)
- `POST /api/keka/sync` endpoint that:
  1. Fetches all employees from Keka (paginate with pageNumber/pageSize, default 100)
  2. Filters to @hoichoi.tv and @svf.in emails only
  3. Upserts into users table (match by email, store keka employee ID as keka_id)
  4. Maps Keka department/job title to the app's role system (designer, video_editor, requester, etc.) - or defaults to 'requester'
  5. Returns sync summary (added, updated, skipped counts)
- Rate limit: 50 requests/minute. Add a delay between paginated calls.
- Add a "Sync from Keka" button in the Settings view
- If Keka credentials are not configured (env vars empty), hide the sync button and skip gracefully

### Phase 5: Email Notifications via Microsoft Graph API

**IMPORTANT**: Do NOT use SMTP/Nodemailer. Microsoft deprecated Basic Auth for SMTP on March 1, 2026. Use Microsoft Graph API with OAuth 2.0 client credentials flow instead.

**Azure AD App Registration needed** (I will do this later):
- Application permission: `Mail.Send` (Application, not Delegated)
- Admin consent granted for the tenant

**Token endpoint**:
```
POST https://login.microsoftonline.com/{AZURE_TENANT_ID}/oauth2/v2.0/token
Content-Type: application/x-www-form-urlencoded

client_id={AZURE_CLIENT_ID}
client_secret={AZURE_CLIENT_SECRET}
scope=https://graph.microsoft.com/.default
grant_type=client_credentials
```

**Send mail endpoint**:
```
POST https://graph.microsoft.com/v1.0/users/{MAIL_FROM}/sendMail
Authorization: Bearer {access_token}
Content-Type: application/json

{
  "message": {
    "subject": "...",
    "body": { "contentType": "HTML", "content": "..." },
    "toRecipients": [{ "emailAddress": { "address": "user@hoichoi.tv", "name": "User Name" } }]
  },
  "saveToSentItems": false
}
```

**Build**:
- `services/email.js` - Microsoft Graph email client with:
  - Token caching (expires_in ~3599 seconds, refresh 5 minutes before expiry)
  - `sendMail(to, subject, htmlBody)` method (accepts array of recipients)
  - Graceful failure: if env vars are missing or token fails, log warning and skip (don't crash the app)
  - All email sending is async, fire-and-forget (don't block API responses)

- `services/email-templates.js` - HTML email templates:
  - Use inline CSS (email clients strip <style> tags)
  - Dark theme matching the app: background #0a0f14, card background #111827, text #e2e8f0, accent #2dd4bf
  - Hoichoi teal CTA button linking to the request in the app
  - Mobile-responsive (max-width: 600px)
  - Templates for:

  **a) Task Assignment**
  - Trigger: When `POST /api/requests/:id/assign` or `POST /api/deliverables/:id/assign` is called
  - To: assignee's email
  - Subject: `[CreativeOps] New task assigned: {request title}`
  - Body: request title, campaign name, asset type, priority badge, deadline, brief summary, CTA button "View Task"

  **b) Status Change**
  - Trigger: When `POST /api/requests/:id/status` is called
  - To: assignee + requester (created_by)
  - Subject: `[CreativeOps] Status update: {title} >> {new status}`
  - Body: old status >> new status with visual indicator, request details, CTA button

  **c) New Comment**
  - Trigger: When `POST /api/requests/:id/comments` is called
  - To: all participants (assignee + requester + previous commenters), excluding the commenter
  - Subject: `[CreativeOps] New comment on: {title}`
  - Body: commenter name, comment text preview, request context, CTA button "Reply"

  **d) Approval Decision**
  - Trigger: When status changes to `final_approved` or gets sent back (status regression)
  - To: assignee
  - Subject: `[CreativeOps] Approved: {title}` or `[CreativeOps] Changes needed: {title}`
  - Body: approver name, any comment, next steps, CTA button

## Critical Constraints

1. **Do NOT modify app.js, style.css, data-service.js, or index.html.** The only frontend file that changes is `storage-client.js`.
2. **storage-client.js must keep the same public API** (method names, arguments, return shapes). It is called by data-service.js which is called by app.js. Changing the interface breaks the whole app.
3. **No TypeScript, no bundler, no build step.** Plain Node.js + Express. CommonJS `require()` everywhere.
4. **Use `pg` package directly** for PostgreSQL. No Sequelize, no Knex, no Prisma.
5. **All external API calls (Keka, Microsoft Graph) must fail gracefully.** If credentials are missing or API is down, log a warning and continue. The app must work without Keka or email configured.
6. **Emails are fire-and-forget.** Never block an API response waiting for email to send.
7. **Use `express-session` with a simple memory store** (fine for internal tool with <20 users).
8. **Seed data from seed-data.js should be used for initial database population** so the app has data on first boot.
